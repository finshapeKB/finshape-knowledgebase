  2. Use case vs Task

    Use case from the user point of view is the specific result that the user want to achieve by making some actions within software product. As far as use case is a term mainly used within software domain, we may use more general term and establish Task entity. Task entity is a set of steps that have to be done to reach a specific result.

    • Task types

      Tasks domains, types and subtypes were defined to ease understanding software product features, and what benefits they provide to a software user. From the business point of view we may divide all possible tasks into separate domains: Functional Tasks and Product tasks. Tasks are divided into types within each domain and may be divided into subtypes.

    • Product tasks

      Product Tasks is a domain that comprises all tasks which manufacturer employees handle to design, produce, sale, deliver and putting its products to use. Product task types are associated with enterprise processes and are divided into types. Each task type may have subtypes.

    • Functional tasks

      Functional Tasks domain comprises all tasks that are related to installing, maintenance and using software products by bank employees and their clients. Functional tasks are handled by bank employees and their clients. Functional domain tasks are divided into seven types, each task type may have subtypes.

      • Admin Tasks: FT01

        Admin Functional tasks are admin Role specific tasks.

      • Security Tasks: FT02

        Security Functional tasks are Security officer Role specific tasks.
