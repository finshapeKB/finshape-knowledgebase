Workspace vs App
Workspace is a desktop (Dashboard) or mobile GUI page that is opened as bank's employees or clients login to their banking application. Workspace may be designed for all Roles at the same time, or for a specific Role to provide handling only Role specific Tasks. Workspace usually comprise Introduction or Overview GUI page. Application from the other hand is a software package that performs a specific functions directly for an end user and comprises all GUI elements: workspace, pages, controls etc.
Workspaces for Bank's Employees
Bank's Employee workspace is a desktop page that is opened as bank's employees log in to their banking application. Bank's Employee manage their banking operations using role rights, so each role may have it's own role specific workspace.
Workspaces for Corporate Client (Business)
Bank's Corporate Client workspace is a desktop or a mobile page that is opened as bank's corporate clients log in to their internet banking application. Corporate clients manage their banking operations using Roles.
Workspaces for Retail Client (Retail)
Workspace for Retail Client is a desktop or a mobile UI page that is opened as Retail Clients login to the internet banking application. Retail Clients manage their banking operations within workspace.
Use case vs Task
Use case from the user point of view is the specific result that the user want to achieve by making some actions within software product. As far as use case is a term mainly used within software domain, we may use more general term and establish Task entity. Task entity is a set of steps that have to be done to reach a specific result.
Task types
Tasks domains, types and subtypes were defined to ease understanding software product features, and what benefits they provide to a software user. From the business point of view we may divide all possible tasks into separate domains: Functional Tasks and Product tasks. Tasks are divided into types within each domain and may be divided into subtypes.
Product tasks
Product Tasks is a domain that comprises all tasks which manufacturer employees handle to design, produce, sale, deliver and putting its products to use. Product task types are associated with enterprise processes and are divided into types. Each task type may have subtypes.
Functional tasks
Functional Tasks domain comprises all tasks that are related to installing, maintenance and using software products by bank employees and their clients. Functional tasks are handled by bank employees and their clients. Functional domain tasks are divided into seven types, each task type may have subtypes.

Data objects
Data object (DO) is an information entity that comprise details about all data and services that are needed to handle specific Step of a Functional task. Data object may be connected to more than one Step at the same time. Many Data objects may be connected to one Functional task as far as task may comprise several Steps.
Data object types
Data object (DO) types are categories that were made by dividing all Functional tasks to an understandable logic groups. Data objects are divided into types and subtypes the same way as Functional tasks and Steps do to simplify their interconnection and relationship understanding.
